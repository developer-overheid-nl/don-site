# Hoe draag ik een kennisbank-artikel aan?

Heb jij een waardevol artikel voor onze kennisbank? Wij plaatsen hem graag.

## Markdown
- Een bijdrage aan onze blog dien je in alle gevallen aan te dragen als Markdown bestand.
- Een markdown bestand heeft de `.md` bestandsextensie.
- De filename van jouw blog ziet er bijvoorbeeld zo uit: `mijn-artikel-over-spring-framework.md`.

Er zijn verschillende manieren om een kennisbank-artikel aan te dragen:

## 1. Stuur ons een e-mail
Stuur een e-mail naar [developer.overheid@geonovum.nl](developer.overheid@geonovum.nl) met het markdown-bestand van jouw blogpost.

## 2. Maak een Gist en dien een issue in
Je kunt een artikel schrijven in Markdown en deze als een [Gist](https://gist.github.com/) opslaan. Zorg ervoor dat je de Gist openbaar maakt en de link naar de Gist deelt in een [issue](https://github.com/developer-overheid-nl/don-site/issues/new).

## 3. Maak een Pull Request
Ook is het mogelijk om een blogpost aan te bieden door middel van een pull request. Let er wel op dat je de blogpost in de draft map zet:

`/blog/draft/example.md`

Als je nog geen lid bent van onze repository, maak dan eerst een [fork](https://github.com/developer-overheid-nl/don-site/fork). Deze fork kun je vervolgens aanbieden als pull request bij ons.
