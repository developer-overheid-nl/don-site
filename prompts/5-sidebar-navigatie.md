### `_category_.json` bestanden voor unieke keys

Omdat meerdere categorieĂ«n dezelfde naam hebben (bv. "Standaarden" komt voor
onder api-ontwikkeling, data, devops, front-end en open-source), vereist
Docusaurus unieke translation keys. Maak voor elke submap met een dubbele naam
een `_category_.json` aan:

```
docs/<thema>/standaarden/_category_.json  â†’  { "label": "Standaarden", "key": "<thema>-standaarden" }
docs/<thema>/tools/_category_.json        â†’  { "label": "Tools", "key": "<thema>-tools" }
docs/<thema>/tutorials/_category_.json    â†’  { "label": "Tutorials", "key": "<thema>-tutorials" }
```

### `docusaurus.config.ts` â€” autoCollapseCategories

Voeg `autoCollapseCategories` toe aan `themeConfig` zodat bij het uitklappen van
een themacategorie de andere automatisch dichtklappen (accordion-gedrag):

```typescript
// In themeConfig:
docs: {
  sidebar: {
    autoCollapseCategories: true,
  },
},
```

### `docusaurus.config.ts` â€” sidebarItemsGenerator

Voeg een `sidebarItemsGenerator` toe aan de docs plugin configuratie. Dit
voorkomt dat index-bestanden als apart kind-item verschijnen onder hun categorie
(ze worden al gebruikt als categorie-link):

```typescript
// In presets > classic > docs:
docs: {
  path: "docs",
  routeBasePath: "kennisbank",
  sidebarPath: "./sidebars.ts",
  tags: "../tags.yml",
  onInlineTags: "throw",
  sidebarItemsGenerator: async function ({
    defaultSidebarItemsGenerator,
    ...args
  }) {
    const items = await defaultSidebarItemsGenerator(args);
    // Verwijder index-docs van het top-niveau â€” ze worden al als category link gebruikt
    return items.filter(
      (item) =>
        !(item.type === "doc" && item.id.endsWith("/index")),
    );
  },
  editUrl: "https://github.com/developer-overheid-nl/don-site/tree/main/",
},
```

### Sidebar-toewijzing per document

- `docs/index.mdx` heeft `displayed_sidebar: kennisbank`
- Alle andere docs hoeven geen `displayed_sidebar` â€” ze worden automatisch
  gevonden via de `autogenerated` items
