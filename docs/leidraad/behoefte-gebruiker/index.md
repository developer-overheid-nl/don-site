Begrijp je gebruikers en hun behoeften. Zorg ervoor dat je inzicht hebt in wat je gebruikers drijft en wat dat betekent voor je project of programma.


## Belangrijke redenen
- Als je niet uitzoekt welke problemen een gebruiker probeert op te lossen kan het zomaar zijn dat je features bouwt zonder dat ze ooit gebruikt gaan worden.
- Een team dat werkt aan software dat ook ooit echt van belang is voor burgers ervaart meer zingeving.
